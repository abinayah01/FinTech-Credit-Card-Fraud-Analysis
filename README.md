# FinTech Credit Card Fraud Analysis Case Study

## 📌 Project Overview
This project involves analyzing financial transaction data to detect and prevent credit card fraud for a FinTech platform. By identifying suspicious login frequencies and unauthorized transaction patterns, this analysis helps risk teams mitigate losses and protect customer accounts.

---

## 📊 Case Study Walkthrough (STAR Method)

### 1. Situation
With the rise of online digital banking, credit card fraud has increased significantly. The fraud risk management team needed a data-driven approach to quickly flag compromised accounts and isolate suspicious user sessions before fraudulent transactions could be completed.

### 2. Task
My primary tasks were to:
* Analyze large transactional and session datasets containing user activity logs.
* Extract key variables such as customer login frequency, transaction amounts, and location data to spot anomalies.
* Create a tracking framework that can successfully identify "who is a fraud" or which patterns indicate highly fraudulent activity.

### 3. Action
* **Data Organization:** Structured the raw financial data by establishing distinct relationships between transactional tables and customer profile logs using unique identifiers like `Customer_ID`.
* **Behavioral Pattern Tracking:** Analyzed the specific frequency of logins within short time frames to detect automated bot attacks or credential stuffing.
* **Metric Extraction:** Grouped high-risk transactions by location mismatches (e.g., logins from two different cities within an hour) and unusually high transaction amounts.

### 4. Result
* Provided the risk team with clear indicators of compromised accounts, enabling faster freezing of fraudulent activities.
* Successfully established a clear profiling system that separates legitimate customer behaviors from high-risk, anomalous patterns.
